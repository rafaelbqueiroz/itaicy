Vamos atacar o problema em duas frentes:

Desativar o Portal do Headless UI/Popper (hoje ele está “teleportando” o menu para fora do <header>, renderizando-o no <body> e aí volta a usar transform: translateY(16px) para posicionar em relação à viewport).

Ancorar o painel ao próprio <header> com CSS puro, via position: relative no wrapper e position: absolute; inset-x-0; top-full; no popover.

Passo 1 – Desative o portal
No seu PopoverContent, passe a prop que desabilita o portal. Exemplo usando Radix UI / Headless UI:

tsx
Copiar
Editar
// antes:
<Popover>
  <PopoverTrigger>…</PopoverTrigger>
  <PopoverContent className="…">…</PopoverContent>
</Popover>

// depois:
<Popover>
  <PopoverTrigger>…</PopoverTrigger>
  <PopoverContent 
    className="absolute inset-x-0 top-full mt-0 z-40"
    portal={false}         // <— desliga o teleport
  >
    …
  </PopoverContent>
</Popover>
Se a sua biblioteca chamar isso de portal={false}, disablePortal, unstable_portal={false} etc, use o nome certo. O importante é que o conteúdo seja renderizado dentro do <header>.

Passo 2 – CSS de ancoragem
No seu arquivo de estilo global ou num módulo CSS:

Editar
/* 1) transforme o header num contêiner relativo */
header.site-header {
  position: relative;
  z-index: 30; /* acima do vídeo */
}

/* 2) o popover deve “colar” à base do header, ocupar toda a largura interna */
header.site-header .popover-content {
  position: absolute !important;
  inset-inline-start: 0;   /* left: 0 */
  inset-inline-end:   0;   /* right: 0 */
  top: 100%;               /* imediatamente abaixo do header */
  margin-top: 0 !important; /* anula utilitários mt-# */
  transform: translateY(0) !important; /* anula qualquer translate */
  background-color: white;
  box-shadow: 0 4px 16px rgba(0,0,0,0.08);
  border-radius: var(--radius-md);
}
Atenção aos seletores:

.site-header deve ser a classe que engloba todo o cabeçalho, do logo ao botão Reservar.

.popover-content (ou o nome que sua lib aplica) deve referenciar o painel do mega-menu.
